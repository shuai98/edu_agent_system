## 项目定位

**项目名称**：EduReflex - 多智能体协作学习系统

**核心目标**：从"被动检索"升级到"主动导学"，通过多个 AI Agent 协作，实现智能化的学习路径规划和知识整合。

**技术定位**：展示 Agent 开发能力、后端工程能力、系统架构思维，用于毕业设计答辩和面试展示。

## 已实现的功能

### 1. 核心架构
- **LangGraph 状态机**：使用 LangGraph 框架构建了支持循环和条件跳转的工作流
- **三个 Agent 节点**：
  - **Planner（规划者）**：接收用户学习目标，拆解为 3-5 个有序子任务，生成搜索关键词
  - **Researcher（研究员）**：并发调用搜索工具和 RAG 系统，收集知识并整合
  - **Critic（反思者）**：评估生成内容的质量，不合格则触发循环重做
- **条件路由**：根据 Critic 的评估结果，动态决定是结束流程还是回到 Researcher 重新生成

### 2. 工具集成
- **DuckDuckGo 搜索**：免费的网络搜索工具，获取最新技术资讯
- **RAG 系统接口**：调用我的第一个项目（RAG 测评系统）作为权威知识源
- **降级策略**：RAG 失败时自动切换到网络搜索，确保系统可用性

### 3. 后端工程能力
- **异步编程**：全链路使用 async/await，通过 asyncio.gather 并发调用工具，性能提升 50%
- **状态持久化**：使用 Redis Checkpointer 保存 Agent 状态，支持断点续行
- **流式输出**：基于 SSE（Server-Sent Events）实时推送 Agent 思考过程和耗时
- **防御性编程**：
  - 启动前检查环境变量
  - 所有外部调用都有异常处理和超时机制
  - 设置最大重试次数（2次），防止死循环
  - 搜索结果截断，防止 Token 溢出

### 4. API 服务
- **FastAPI 框架**：异步高性能 Web 框架
- **主要接口**：
  - `POST /api/query`：主查询接口，支持流式和非流式两种模式
  - `GET /api/history/{thread_id}`：查询会话历史
- **CORS 支持**：允许前端跨域访问
- **自动文档**：Swagger UI（http://localhost:8001/docs）

### 5. 前端展示
- **Web 界面**：HTML + JavaScript 实现的聊天式界面
- **Markdown 渲染**：使用 marked.js 渲染 Agent 生成的学习内容
- **实时反馈**：显示每个节点的执行状态和耗时
- **用户体验**：类似 ChatGPT 的流式输出效果

### 6. 文档体系
- **README.md**：项目概述、技术架构、快速开始
- **QUICKSTART.md**：详细的安装步骤、常见问题排查
- **INTERVIEW_GUIDE.md**：面试准备手册，包含核心概念讲解、常见问题答案、技术话术
- **ARCHITECTURE.md**：系统架构图、数据流转图、性能指标
- **TODO.md**：项目检查清单、下一步行动计划

## 核心设计亮点

### 1. 状态机 vs 简单链
**传统 LangChain Chain（线性）**：
```
输入 → LLM → 输出
```
- 只能顺序执行
- 无法根据结果调整策略

**LangGraph 状态机（图）**：
```
输入 → Planner → Researcher → Critic
                     ↑            ↓
                     └─── 不合格 ──┘
```
- 支持条件跳转和循环
- 可以根据 Critic 评估动态决定下一步
- 状态可持久化，支持断点续行

### 2. 自我反思机制
- Critic 节点使用 LLM 评估内容的完整性、准确性、逻辑性
- 不合格时设置 `revision_needed=True`，触发循环边
- 设置最大重试次数，防止无限循环
- 这是抑制 LLM 幻觉的有效手段

### 3. 异步并发优化
**优化前（串行）**：
```python
result1 = await search_web(query)    # 3秒
result2 = await query_rag(query)     # 3秒
# 总耗时：6秒
```

**优化后（并发）**：
```python
results = await asyncio.gather(
    search_web(query),
    query_rag(query),
)
# 总耗时：3秒（取最慢的）
```

### 4. 微服务联动
- Agent 可以调用我的第一个项目（RAG 系统）作为知识工具
- 展示了系统集成能力和架构思维
- 避免重复造轮，复用已有能力

### 5. 流式输出体验
- 使用 SSE 协议实时推送节点状态
- 前端显示每个节点的执行进度和耗时
- 类似 ChatGPT 的打字效果，提升用户感知

## 实际运行效果

### 测试案例
**用户输入**："帮我掌握 Python 异步编程"

**系统执行流程**：
1. **Planner 节点**（8.5秒）
   - 拆解为 3 个子任务：理解协程概念、掌握 async/await 语法、实战异步请求
   - 生成搜索关键词："Python asyncio 协程 事件循环"

2. **Researcher 节点**（3.2秒）
   - 并发调用 DuckDuckGo 搜索和 RAG 系统
   - 整合多源信息，生成初步学习材料

3. **Critic 节点**（16.8秒）
   - 评估内容质量：完整性、准确性、逻辑性、实用性
   - 判断为合格，通过审核

4. **最终输出**（总耗时 28.5秒）
   - 生成完整的学习材料，包含：
     - 协程的概念定义
     - 与线程的区别
     - async/await 语法示例
     - 事件循环原理
     - 实战代码示例

## 当前的不足与限制

### 1. 功能层面
- ❌ **没有用户认证**：所有人都可以访问，无法区分用户
- ❌ **没有历史记录查询**：虽然有接口，但前端未实现
- ❌ **没有长期记忆**：无法记住用户的学习偏好和薄弱环节
- ❌ **只处理单个任务**：不支持多任务并行（如同时学习多个知识点）
- ❌ **Prompt 未充分优化**：Planner 和 Critic 的提示词还有改进空间

### 2. 工程层面
- ❌ **没有并发测试**：未使用 Locust 等工具进行压力测试
- ❌ **没有监控告警**：缺少 Prometheus + Grafana 监控
- ❌ **没有日志分析**：未集成 ELK Stack
- ❌ **没有单元测试覆盖**：缺少 pytest 测试用例
- ❌ **没有 CI/CD**：未配置自动化部署流程

### 3. 安全层面
- ❌ **没有 API Key 认证**：任何人都可以调用接口
- ❌ **没有请求频率限制**：可能被恶意刷接口
- ❌ **没有输入验证**：未对用户输入进行充分清洗
- ❌ **环境变量暴露风险**：.env 文件需要妥善保管

### 4. 用户体验层面
- ❌ **前端较简陋**：只是一个 Demo，未使用 React/Vue 等现代框架
- ❌ **没有移动端适配**：只在桌面浏览器测试过
- ❌ **没有错误重试**：网络失败时用户需要手动重新提问
- ❌ **没有进度条**：只有文字提示，缺少可视化进度

### 5. 评测层面
- ❌ **没有效果评测**：未使用 Ragas 等工具量化评估 Agent 效果
- ❌ **没有对比实验**：未对比"RAG 直接生成"vs"Agent 协作生成"的差异
- ❌ **没有用户反馈机制**：无法收集用户满意度

## 技术债务与已知问题

### 1. 依赖问题
- DuckDuckGo 搜索在国内可能被墙，需要代理
- 搜索结果质量不稳定，有时返回空结果
- RAG 系统需要单独启动，增加了部署复杂度

### 2. 性能瓶颈
- LLM API 调用是主要耗时（占 80%+）
- 单次查询需要 20-30 秒，用户可能觉得慢
- 没有缓存机制，相同问题每次都重新生成

### 3. 可靠性问题
- 网络请求失败时，整个流程会中断
- Critic 的评估标准不够稳定，有时会误判
- 最大重试次数设置为 2，可能导致质量不够高就被强制通过
