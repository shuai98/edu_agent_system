## 🎯 核心面试问题准备

### 1. 项目介绍（1 分钟）

**模板**：
```
我开发了一个基于 LangGraph 的多智能体学习系统。

核心亮点：
1. 三个 Agent 协作：Planner 规划、Researcher 搜索、Critic 审核
2. 自我反思机制：内容不合格会自动重做
3. 异步并发：性能提升 50%
4. 流式输出：实时展示思考过程

技术栈：LangGraph、FastAPI、DeepSeek-V3、Redis

这个项目展示了我在 Agent 系统设计、异步编程、
状态管理等方面的能力。
```

### 2. 为什么用 LangGraph？

**答案**：
```
LangGraph 支持循环和条件跳转，可以实现自我反思。
传统 LangChain 的 Chain 只能线性执行，
无法根据 Critic 的评估结果动态调整。

这是实现 Agent 自主决策的关键。
```

### 3. 异步并发如何实现？

**答案**：
```
使用 asyncio.gather 并发调用搜索和 RAG。
原本串行需要 6 秒（3+3），并发后只需 3 秒。

关键是所有 I/O 操作都用 async/await，
不阻塞事件循环。
```

### 4. 如何防止死循环？

**答案**：
```
两层保护：
1. 最大重试次数（revision_count <= 2）
2. 每个节点有超时限制

即使 Critic 一直不满意，也会强制通过。
```

### 5. 如何评估效果？

**答案**：
```
三个维度：
1. 内容质量：对比 RAG vs Agent
2. 响应速度：记录端到端延迟
3. 用户满意度：A/B 测试

我准备了多个测试案例，有详细的运行日志。
```

## 🚨 常见问题应对

### Q: 为什么不用 AutoGPT/LangChain？
A: LangGraph 更适合复杂工作流，支持循环和条件跳转。

### Q: 如何处理并发请求？
A: FastAPI 异步 + thread_id 会话隔离。

### Q: 遇到的最大挑战？
A: 调试条件边的循环逻辑，通过日志和最大重试解决。

### Q: 如何优化性能？
A: 异步并发、图预编译、结果缓存。

### Q: 如何保证质量？
A: Critic 节点评估、最大重试、详细日志。
