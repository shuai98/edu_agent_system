# EduReflex 项目简历描述（仿照格式）

---

## 基于多智能体协作的自主学习导航系统

**项目时间**：2025年12月 - 2026年03月  
**技术栈**：Python, LangGraph, FastAPI, DeepSeek-V3, Redis, SSE, httpx, DuckDuckGo

---

### 项目描述

针对传统 RAG 系统在复杂学习场景下"检索即返回"的局限性，设计并开发了一套基于多智能体协作的自主学习导航系统。该系统通过 LangGraph 状态机实现了从"被动问答"到"主动规划-执行-反思"的范式升级，利用大模型在任务拆解、多源融合和质量把控方面的能力，为学生提供系统化的学习路径规划和知识整合服务。

---

### 核心工作

**架构设计与多体协同**：基于 LangGraph 构建支持循环和条件跳转的有向有环图（DAG with Cycles）架构，设计了 Planner（任务规划）、Researcher（知识检索）、Critic（质量审核）三个 Agent 节点。针对复杂学习目标（如"掌握 Python 异步编程"），Planner 自动拆解为 3-5 个有序子任务，Researcher 并发调用搜索引擎和 RAG 系统收集知识，Critic 通过 LLM 评估内容质量并触发循环重做，实现"规划→执行→反思"的闭环。

**工具调用与微服务联动**：基于 Function Calling 技术将 DuckDuckGo 搜索和自研 RAG 测评系统封装为标准化工具接口，Researcher 节点根据任务类型自主选择调用策略。利用 httpx 异步客户端实现跨服务通信，将第一个项目（RAG 测评系统）作为知识工具集成，展示了微服务架构的系统集成能力。设计降级策略，当 RAG 系统不可用时自动切换到网络搜索，系统可用性达 99.5%。

**异步并发与性能优化**：全链路采用 Async/Await 实现 LLM 请求和外部 API 调用的非阻塞处理，在大并发场景下显著提升系统吞吐量。利用 asyncio.gather 并发调用搜索引擎和 RAG 接口，将原本串行 6 秒的 I/O 操作压缩至 3 秒，性能提升约 50%。

**状态持久化与可靠性**：基于 Redis 实现 Checkpointer 机制，支持 Agent 执行状态的序列化存储与断点续行。通过 thread_id 实现会话级隔离，保证多用户并发场景下的状态一致性。设计自我反思循环，Critic 节点评估内容的完整性、准确性和逻辑性，不合格时触发重做（最大重试 2 次），相比直接 LLM 生成，事实性错误率降低约 35%。

**流式传输与用户体验**：基于 FastAPI 搭建异步 API 服务，利用 SSE（Server-Sent Events）实现实时流式传输。前端可逐节点展示 Agent 思考过程（"正在规划..." → "正在检索..." → "正在审核..."），首字响应时间优化至 2 秒内，提升用户感知体验。开发 Web 前端界面，支持 Markdown 渲染和耗时统计，便于技术演示和答辩展示。

---

### 技术成果

- **性能指标**：端到端平均响应时间 28 秒，首字响应 2 秒，并发支持 10+ QPS（单实例）
- **质量提升**：通过自我反思机制，幻觉率相比直接生成降低 35%（基于 10 个测试案例）
- **系统稳定性**：通过降级策略和异常处理，系统可用性达 99.5%
- **代码规模**：核心代码 2000+ 行，模块化设计（nodes/tools/api 三层架构）
- **文档完整性**：编写 README、架构文档、面试指南等 5 份技术文档，代码注释覆盖率 > 80%
