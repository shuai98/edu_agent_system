## 二、数据流转图

### 2.1 正常流程（一次通过）

```
用户输入: "帮我掌握 Python 异步编程"
    │
    ▼
┌─────────────────────────────────────────┐
│ Planner 节点                             │
│ - 输入: user_input                       │
│ - 处理: 调用 LLM 拆解任务                │
│ - 输出: plan (JSON 格式的子任务列表)     │
│         search_queries (搜索关键词)      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│ Researcher 节点                          │
│ - 输入: search_queries                   │
│ - 处理: 并发调用工具                     │
│   ├─ search_web("Python asyncio")       │
│   └─ query_rag("Python asyncio")        │
│ - 输出: search_results, rag_results     │
│         final_answer (初步答案)          │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│ Critic 节点                              │
│ - 输入: final_answer                     │
│ - 处理: 调用 LLM 评估质量                │
│ - 输出: critique (评价)                  │
│         revision_needed = False          │
└─────────────────┬───────────────────────┘
                  │
                  ▼
              返回结果
```


### 2.2 循环流程（需要重做）

```
┌─────────────────────────────────────────┐
│ Critic 节点                              │

                  │
                  ▼
              返回结果
```

## 四、技术栈详解

### 4.1 核心框架

| 组件 | 版本 | 作用 | 为什么选择 |
|------|------|------|-----------|
| LangGraph | 0.2.45 | 状态机编排 | 支持循环、条件跳转，是 Agent 开发的最佳框架 |
| LangChain | 0.3.7 | 基础组件 | LangGraph 的依赖，提供 LLM 抽象 |
| FastAPI | 0.115.4 | Web 框架 | 异步高性能，自动生成 API 文档 |
| Redis | 5.2.0 | 状态存储 | 持久化 Checkpointer，支持分布式 |


### 4.2 模型选择

| 模型 | 价格 | 速度 | 能力 | 推荐场景 |
|------|------|------|------|---------|
| DeepSeek-V3 | ¥0.001/1K tokens | 快 | 推理强 | 学生项目、国内部署 |
| GPT-4o | ¥0.015/1K tokens | 中 | 全能 | 生产环境、高质量要求 |


### 4.3 搜索工具

| 工具 | 费用 | 质量 | 限制 | 推荐场景 |
|------|------|------|------|---------|
| DuckDuckGo | 免费 | 中 | 可能被墙 | 开发测试 |
| Tavily | 付费 | 高 | 需要 API Key | 生产环境 |


## 五、部署架构

### 5.1 开发环境

```
本地机器
├── Python 3.9+
├── Redis (可选，不装则用内存模式)
└── EduReflex
    ├── FastAPI 服务 (端口 8001)
    └── 前端页面 (frontend_demo.html)
```


### 5.2 生产环境（推荐）

```
云服务器 (阿里云/腾讯云)
├── Docker 容器
│   ├── EduReflex API (多实例)
│   ├── Redis (持久化)
│   └── Nginx (负载均衡)
├── 监控
│   ├── Prometheus (指标采集)
│   └── Grafana (可视化)
└── 日志
    └── ELK Stack (日志分析)
```


## 六、性能指标

### 6.1 响应时间

| 场景 | 平均耗时 | 说明 |
|------|---------|------|
| Planner 节点 | 2-3 秒 | LLM 生成规划 |
| Researcher 节点 | 3-5 秒 | 并发调用工具 |
| Critic 节点 | 2-3 秒 | LLM 评估质量 |
| 总耗时（一次通过） | 7-11 秒 | 端到端 |
| 总耗时（需要重做） | 12-18 秒 | 多一次循环 |


### 6.2 并发能力

| 指标 | 数值 | 说明 |
|------|------|------|
| 单实例 QPS | 10-20 | 受 LLM API 限制 |
| 多实例 QPS | 100+ | 水平扩展 |
| 内存占用 | 200-500 MB | 单实例 |
| CPU 占用 | 10-30% | 单实例 |

## 十、总结

EduReflex 是一个完整的多智能体协作系统，展示了：

1. **技术深度**：LangGraph 状态机、异步编程、流式输出
2. **工程能力**：持久化、降级策略、防御性编程
3. **系统思维**：微服务联动、性能优化、可扩展性
4. **创新性**：自我反思机制、跨源知识融合

这个项目不仅是一个毕业设计，更是一个可以在面试中充分展示能力的作品集。
