## ✅ 改进 1：使用结构化输出优化 Prompt

### 改进内容

**文件**: `app/nodes/planner.py`

**核心变化**:
1. 使用 Pydantic 模型定义输出格式
2. 使用 `with_structured_output()` 方法
3. 添加多个 Few-shot 示例
4. 实现三层降级策略

### 技术亮点

```python
# 定义结构化模型
class Task(BaseModel):
    id: int
    title: str
    description: str
    search_query: str

class LearningPlan(BaseModel):
    goal: str
    tasks: List[Task]

# 使用结构化输出
llm_with_structure = llm.with_structured_output(LearningPlan)
plan: LearningPlan = await llm_with_structure.ainvoke(messages)
```

### 优势

1. **避免 JSON 解析错误**: 直接返回 Pydantic 对象，类型安全
2. **更稳定的输出**: LangChain 会自动处理格式问题
3. **更好的提示**: 增加了 3 个 Few-shot 示例
4. **降级策略**: 结构化失败 → 传统 JSON → 简化模式

1. 类型安全：直接返回 Python 对象，避免 JSON 解析错误
2. 自动验证：Pydantic 会验证字段类型和必填项
3. 更稳定：LangChain 会自动处理格式问题

我还实现了三层降级策略，确保即使结构化输出失败，
